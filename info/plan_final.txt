Plan: Fraction Wheel — Roadmap и требования (RU)
TL;DR - собрать расширяемую мини‑игру для детей (~10 лет) по сложению дробей: адаптивный генератор задач, многослойная подсказка, отдельный блок обучения про общий знаменатель, модуль аналитики (по дробям и по датам), TTS и анимации с опцией выключения.

Steps


Дизайн данных и архитектуры: STATE (задача, selected, settings, analytics), события аналитики, формат хранения (localStorage → later server). parallel with step 3
Верстка минимального HTML/CSS шаблона и компонент колеса: статический рендер, центрированный flex layout, SVG‑сектора. depends on step 2
Взаимодействие: клики по секторам → обновление STATE.selected; toggle подсказки; кнопки Проверить! и Следующая.
Подсказка: реализовать как многошаговый блок (1. найти общий знаменатель, 2. привести дроби, 3. сложить) с возможностью показать/скрыть; первая подсказка показывается автоматически при первом запуске.
Блок «Общий знаменатель»: отдельный визуальный урок (интерактивные шаги + TTS). parallel with step 4/5
Настройки: переключатели для анимаций, озвучки, количества задач, режима (ручной/умный), экспорт/очистка аналитики.
Генератор задач: набор шаблонов + регулировка количества задач + «умный режим» (базовая модель: per‑fraction mastery score; выбирать задачи с более высокой ошибочностью). depends on analytics model
Модуль аналитики: трекать события — attempt, correct, timeSpent, date, fractionKey; агрегаты: ошибок по дробям, прогресс по дням. Хранение: localStorage JSON, экспорт CSV. parallel with step 8
TTS: использовать Web Speech API как опцию; настройки выбора голоса и скорость.
Анимации: CSS transitions for sector click; глобальный toggle в settings.
Тесты: unit tests (Jest) для генератора задач, аналитики, и логики проверки; e2e (Puppeteer) для ключевых сценариев UI.
Демо/развёртывание: собрать минимальный рабочий билд, загрузить на CodePen/gh‑pages по желанию.

Verification

Unit tests: генератор задач, проверка ответов, аналитика (Jest). 5+ тестов минимум.
E2E: сценарии выбора секторов, подсказка toggle, TTS on/off (Puppeteer/manual).
Accessibility: Lighthouse (целевой минимум 90), aria‑labels, keyboard navigation, screen‑reader smoke test.
Responsive checks: widths 320px..414px (iPhone SE → iPhone 12).
Manual QA: проверить анимации on/off, TTS on/off, экспорт CSV, аналитика по датам.
Decisions / Предложения

Хранение аналитики: стартуем с localStorage (простота, конфиденциальность), позже добавить серверный бекенд/экспорт.
TTS: использовать Web Speech API (кроссбраузерно, без внешних сервисов).
Адаптивность и доступность — приоритеты: flex‑центрирование, SVG + <text> для центра, pointer‑events для кликабельных секторов.
